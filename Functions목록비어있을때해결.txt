==========================================
Vercel Functions 목록이 비어있을 때 해결 방법
==========================================

Functions 목록에 아무것도 없다는 것은 API 파일들이 배포되지 않았다는 의미입니다.

==========================================
가장 흔한 원인
==========================================

1. API 파일들이 GitHub에 업로드되지 않음 (90% 확률)
2. vercel.json이 GitHub에 없음
3. .gitignore에 api 폴더가 제외되어 있음
4. Vercel 배포 설정 문제

==========================================
1단계: GitHub에서 확인 (가장 중요!)
==========================================

1. GitHub 저장소 접속
   - https://github.com/your-username/weather-app

2. 다음 파일들이 있는지 확인:
   □ api/auth/register.js
   □ api/auth/login.js
   □ api/weather.js
   □ api/forecast.js
   □ api/favorites/index.js
   □ api/favorites/[id].js
   □ vercel.json (루트에)

3. 파일이 없다면:
   → GitHub Desktop에서 확인
   → 파일을 추가하고 커밋/푸시 필요

==========================================
2단계: GitHub Desktop에서 확인
==========================================

1. GitHub Desktop 실행
2. 프로젝트 선택
3. 좌측 파일 목록에서 확인:
   - "api" 폴더가 보이는가?
   - "vercel.json" 파일이 보이는가?

4. 파일이 보이지 않거나 "M" (Modified) 표시가 있다면:
   - 파일 선택
   - 하단 "Summary"에 설명 입력
   - "Commit to main" 클릭
   - "Push origin" 클릭

==========================================
3단계: .gitignore 확인
==========================================

1. 프로젝트 폴더에서 .gitignore 파일 열기
2. 다음 줄이 있는지 확인:
   ❌ api/
   ❌ api/**
   ❌ *.js

만약 있다면:
   - 이 줄들을 삭제하거나
   - !api/ 를 추가하여 api 폴더는 포함되도록 설정

==========================================
4단계: Vercel 배포 로그 확인
==========================================

1. Vercel → 프로젝트 → "Deployments"
2. 최신 배포 클릭
3. "Build Logs" 확인

다음 메시지를 찾으세요:
- "Building api/auth/register.js"
- "Compiled successfully"

만약 이런 메시지가 없다면:
→ API 파일이 배포에 포함되지 않았음
→ GitHub에 파일이 없는 것

==========================================
5단계: 강제 재배포
==========================================

1. GitHub에 모든 파일 푸시 확인
2. Vercel → "Deployments"
3. 최신 배포 옆 "..." 메뉴
4. "Redeploy" 클릭
5. ⚠️ 중요: "Use existing Build Cache" 체크 해제
6. "Redeploy" 클릭
7. 배포 완료 대기 (2-3분)

==========================================
6단계: Functions 목록 다시 확인
==========================================

1. Vercel → "Settings" → "Functions"
2. 다음 함수들이 나타나는지 확인:
   - api/auth/register
   - api/auth/login
   - api/weather
   - api/forecast
   - api/favorites/index
   - api/favorites/[id]

여전히 비어있다면:
→ GitHub에 파일이 없는 것이 확실함
→ 1단계로 돌아가서 다시 확인

==========================================
체크리스트
==========================================

□ GitHub 저장소에 api 폴더가 있는가?
□ GitHub에 api/auth/register.js 파일이 있는가?
□ GitHub에 vercel.json 파일이 있는가?
□ .gitignore에 api 폴더가 제외되어 있지 않은가?
□ GitHub Desktop에서 모든 파일이 커밋되었는가?
□ GitHub에 푸시가 완료되었는가?
□ Vercel에서 재배포를 실행했는가?
□ "Use existing Build Cache"를 해제했는가?

==========================================
빠른 해결 방법
==========================================

1. GitHub Desktop 열기
2. 모든 변경사항 확인
3. "Summary"에 입력: "API 파일 추가"
4. "Commit to main" 클릭
5. "Push origin" 클릭
6. Vercel에서 "Redeploy" (캐시 해제)
7. Functions 목록 확인

==========================================
여전히 작동하지 않는다면
==========================================

1. GitHub 저장소 URL 확인
2. Vercel 프로젝트 설정에서:
   - "Settings" → "Git"
   - 연결된 저장소가 올바른지 확인
3. 새로 배포:
   - "Deployments" → "Create Deployment"
   - GitHub 브랜치 선택
   - "Deploy" 클릭
