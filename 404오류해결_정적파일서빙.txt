==========================================
404 오류 해결: 정적 파일 서빙 설정 추가
==========================================

문제: vercel.json에 정적 파일(HTML, CSS, JS) 서빙 설정이 없었습니다.

해결: vercel.json에 "filesystem" 핸들러를 추가했습니다.

==========================================
수정된 vercel.json
==========================================

{
  "version": 2,
  "builds": [
    { "src": "api/**/*.js", "use": "@vercel/node" }
  ],
  "routes": [
    { "src": "/api/(.*)", "dest": "/api/$1" },
    { "handle": "filesystem" },  ← 이 줄 추가!
    { "src": "/(.*)", "dest": "/$1" }
  ]
}

==========================================
다음 단계
==========================================

1. ✅ vercel.json 수정 완료
2. ✅ GitHub에 푸시 완료
3. ⏳ Vercel 자동 재배포 대기 (1-2분)
4. ⏳ 배포 완료 후 사이트 접속 확인

==========================================
확인 방법
==========================================

1. Vercel 대시보드에서 새 배포 확인
2. 배포 완료 후 사이트 접속:
   https://weather-app-tau-lac-75.vercel.app

3. 예상 결과:
   ✅ index.html이 정상적으로 표시됨
   ✅ 404 오류가 사라짐

==========================================
여전히 404가 나타나면
==========================================

1. Vercel → "Deployments" → 최신 배포
2. "Build Logs" 확인
3. 에러 메시지 확인
4. "Runtime Logs" 확인
