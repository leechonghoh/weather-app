# 배포 체크리스트 및 사용자 가이드

실제 사용자들이 사용할 수 있도록 배포하는 전체 과정입니다.

## 📋 배포 전 체크리스트

### 1단계: Supabase 설정 ✅

- [ ] Supabase 계정 생성
- [ ] 새 프로젝트 생성
- [ ] `supabase/schema.sql` 파일 내용을 SQL Editor에서 실행
- [ ] API 키 확인 (Project URL, anon key, service_role key)

### 2단계: GitHub 저장소 준비 ✅

- [ ] GitHub 저장소 생성
- [ ] 로컬 코드를 GitHub에 푸시
- [ ] `.gitignore` 파일이 올바르게 설정되어 있는지 확인 (API 키가 포함된 파일 제외)

### 3단계: Vercel 배포 ✅

- [ ] Vercel 계정 생성 (GitHub 연동)
- [ ] GitHub 저장소를 Vercel에 연결
- [ ] 프로젝트 배포
- [ ] 환경 변수 설정:
  - `SUPABASE_URL`
  - `SUPABASE_SERVICE_KEY`
  - `OPENWEATHER_API_KEY`

### 4단계: 테스트 ✅

- [ ] 배포 URL 접속 확인
- [ ] 회원가입 기능 테스트
- [ ] 로그인 기능 테스트
- [ ] 날씨 검색 기능 테스트
- [ ] 즐겨찾기 추가/삭제 테스트
- [ ] 일주일 예보 기능 테스트

## 🚀 배포 후 사용자 가이드

### 사용자에게 전달할 정보

#### 1. 접속 주소
```
https://your-project-name.vercel.app
```

#### 2. 사용 방법

**처음 사용하는 사용자:**

1. **회원가입**
   - 우측 상단 "로그인" 클릭
   - "회원가입" 링크 클릭
   - 이메일, 비밀번호(6자 이상), 이름(선택사항) 입력
   - "회원가입" 버튼 클릭

2. **로그인**
   - 우측 상단 "로그인" 클릭
   - 이메일과 비밀번호 입력
   - "로그인" 버튼 클릭

3. **날씨 검색**
   - 메인 페이지에서 도시 이름 입력 (예: "서울", "New York")
   - 검색 결과에서 원하는 도시 선택
   - 현재 날씨 정보 확인

4. **즐겨찾기 추가**
   - 날씨 정보가 표시된 상태에서 별 아이콘 클릭
   - 즐겨찾기에 추가됨

5. **즐겨찾기 확인**
   - 상단 메뉴에서 "즐겨찾기" 클릭
   - 추가한 도시들의 날씨 정보를 한눈에 확인
   - 개별 새로고침 또는 전체 새로고침 가능

6. **일주일 예보 확인**
   - 날씨 정보 하단의 "일주일 날씨 예보" 버튼 클릭
   - 카드형 또는 그래프형으로 5일 예보 확인

## 🔧 문제 해결 가이드

### 사용자가 겪을 수 있는 문제들

#### 1. 회원가입/로그인 오류

**증상**: "이미 등록된 이메일입니다" 또는 "이메일 또는 비밀번호가 올바르지 않습니다"

**해결방법**:
- 이메일 주소를 정확히 입력했는지 확인
- 비밀번호가 6자 이상인지 확인
- 이미 가입한 계정이라면 로그인 시도
- 비밀번호를 잊어버렸다면 Supabase 대시보드에서 비밀번호 재설정 (향후 구현 가능)

#### 2. 날씨 정보가 표시되지 않음

**증상**: 검색 후 "도시를 찾을 수 없습니다" 또는 빈 화면

**해결방법**:
- 도시 이름의 철자를 확인
- 영어 도시는 영어로, 한국 도시는 한글로 입력
- 예: "New York" (O), "뉴욕" (X)
- 예: "서울" (O), "Seoul" (O)

#### 3. 즐겨찾기가 저장되지 않음

**증상**: 즐겨찾기를 추가했는데 목록에 나타나지 않음

**해결방법**:
- 로그인이 되어 있는지 확인 (우측 상단에 이름이 표시되어야 함)
- 브라우저를 새로고침 (F5)
- 다시 로그인 후 시도
- 문제가 계속되면 관리자에게 문의

#### 4. 페이지가 로드되지 않음

**증상**: "404 Not Found" 또는 빈 화면

**해결방법**:
- 올바른 URL인지 확인
- 브라우저 캐시 삭제 후 다시 접속
- 다른 브라우저에서 시도
- 인터넷 연결 확인

## 🔐 보안 고려사항

### 현재 보안 설정

1. **API 키 보호**: OpenWeatherMap API 키는 서버에서만 사용 (클라이언트에 노출되지 않음)
2. **데이터베이스 보안**: Row Level Security (RLS) 활성화로 사용자는 자신의 데이터만 접근
3. **인증 토큰**: JWT 토큰을 사용하여 사용자 인증

### 추가 권장사항

1. **HTTPS**: Vercel은 자동으로 HTTPS 제공 (확인 필요)
2. **비밀번호 정책**: 현재 6자 이상, 필요시 더 강화 가능
3. **API Rate Limiting**: 필요시 추가 구현
4. **이메일 인증**: Supabase에서 이메일 인증 활성화 가능
