==========================================
최종 가이드: 처음부터 끝까지
==========================================

이 파일 하나만 따라하면 됩니다!

==========================================
필요한 도구 (모두 무료)
==========================================

1. GitHub - 코드 저장소
2. Vercel - 웹사이트 배포
3. Supabase - 회원가입/로그인 + 데이터베이스
4. OpenWeatherMap - 날씨 정보

==========================================
전체 과정 요약
==========================================

1. GitHub 계정 만들기 → 저장소 생성
2. Supabase 계정 만들기 → 프로젝트 생성 → API 키 복사
3. OpenWeatherMap 계정 만들기 → API 키 복사
4. Vercel 계정 만들기
5. 코드를 GitHub에 업로드
6. Vercel에서 배포 → 환경 변수 설정
7. 테스트

==========================================
상세 가이드
==========================================

자세한 내용은 다음 파일을 참고하세요:

1. "완전초보자용_처음부터배포까지.txt"
   → 단계별 상세 설명

2. "단계별체크리스트.txt"
   → 각 단계별 체크리스트

==========================================
중요 사항
==========================================

⚠️ Supabase 설정:
   - "Enable email confirmations" 반드시 해제!
   - service_role key 사용 (anon key 아님)

⚠️ Vercel 환경 변수:
   - SUPABASE_URL (Project URL)
   - SUPABASE_SERVICE_KEY (service_role key)
   - OPENWEATHER_API_KEY

⚠️ GitHub 업로드:
   - api 폴더 전체
   - vercel.json 파일
   - 모든 HTML, CSS, JS 파일

==========================================
문제 발생 시
==========================================

1. "완전초보자용_처음부터배포까지.txt"의 "문제 해결" 섹션 참고
2. Vercel 배포 로그 확인
3. Supabase 설정 확인
4. 환경 변수 확인
