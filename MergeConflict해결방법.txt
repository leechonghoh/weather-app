==========================================
Merge Conflict 해결 방법
==========================================

"Unexpected token '<'" 오류는 파일에 merge conflict 마커가 있을 때 발생합니다.

==========================================
문제 확인
==========================================

다음 문구가 파일에 있으면 문제입니다:
   <<<<<<< Updated upstream
   =======
   >>>>>>> Stashed changes

이 문구들을 모두 제거해야 합니다!

==========================================
해결 방법 1: GitHub Desktop 사용 (가장 쉬움)
==========================================

1. GitHub Desktop 실행
2. 프로젝트 선택
3. 변경사항 확인
4. merge conflict가 있는 파일이 빨간색으로 표시됨
5. 파일 클릭하여 열기
6. GitHub Desktop이 자동으로 해결 방법 제시:
   - "Use ours" (위쪽 버전 사용)
   - "Use theirs" (아래쪽 버전 사용)
   - "Use both" (둘 다 사용)
7. "Mark as resolved" 클릭
8. 커밋 및 푸시

==========================================
해결 방법 2: 파일 직접 수정
==========================================

1. 문제가 있는 파일 열기
2. 다음 문구들을 찾아서 모두 삭제:
   <<<<<<< Updated upstream
   =======
   >>>>>>> Stashed changes
3. 중복된 내용이 있다면 하나만 남기기
4. 파일 저장
5. GitHub Desktop에서 커밋 및 푸시

==========================================
현재 문제가 있는 파일
==========================================

다음 파일들을 확인하고 수정하세요:

□ package.json (이미 수정됨)
□ 배포가이드_완벽버전.txt (이미 수정됨)
□ 간단배포체크리스트.txt (이미 수정됨)
□ 기타 .txt 파일들

==========================================
빠른 해결 방법
==========================================

1. GitHub Desktop 실행
2. 모든 변경사항 확인
3. merge conflict가 있는 파일 확인
4. 각 파일에서 conflict 마커 제거
5. 커밋 및 푸시

==========================================
주의사항
==========================================

- package.json 파일은 반드시 올바른 JSON 형식이어야 합니다
- merge conflict 마커를 제거하지 않으면 배포가 실패합니다
- 모든 .txt 파일의 conflict도 해결하는 것을 권장합니다
