==========================================
완전 초보자용: 처음부터 배포까지 완벽 가이드
==========================================

목표: 온라인에서 누구나 사용할 수 있는 날씨 앱 만들기
- 여러 지역 날씨 검색
- 즐겨찾기 지역을 계정별로 저장
- 회원가입/로그인 기능

==========================================
필요한 도구 (모두 무료!)
==========================================

1. GitHub (코드 저장소)
   - 용도: 코드를 저장하고 관리
   - 비용: 무료
   - 웹사이트: https://github.com

2. Vercel (웹사이트 배포)
   - 용도: 웹사이트를 온라인에 배포
   - 비용: 무료
   - 웹사이트: https://vercel.com

3. Supabase (회원가입/로그인 + 데이터베이스)
   - 용도: 사용자 계정 관리 및 즐겨찾기 저장
   - 비용: 무료
   - 웹사이트: https://supabase.com

4. OpenWeatherMap (날씨 정보)
   - 용도: 날씨 데이터 제공
   - 비용: 무료 (하루 60회 호출)
   - 웹사이트: https://openweathermap.org

==========================================
1단계: GitHub 계정 만들기 및 저장소 생성
==========================================

1-1. GitHub 계정 만들기
   - https://github.com 접속
   - "Sign up" 클릭
   - 이메일, 비밀번호 입력
   - 사용자 이름 입력
   - 이메일 인증 완료

1-2. 새 저장소 만들기
   - GitHub 로그인 후
   - 우측 상단 "+" 버튼 → "New repository" 클릭
   - Repository name: weather-app (또는 원하는 이름)
   - Description: (선택사항) 날씨 앱
   - Public 선택 (무료 사용을 위해)
   - "Create repository" 클릭

1-3. GitHub Desktop 설치 (선택사항, 권장)
   - https://desktop.github.com/ 접속
   - "Download for Windows" 클릭
   - 설치 프로그램 실행
   - 설치 완료 후 GitHub Desktop 실행
   - GitHub 계정으로 로그인

완료 체크: □ GitHub 계정 생성됨
           □ 저장소 생성됨
           □ GitHub Desktop 설치됨 (선택사항)

==========================================
2단계: Supabase 계정 만들기 및 프로젝트 생성
==========================================

2-1. Supabase 계정 만들기
   - https://supabase.com 접속
   - "Start your project" 클릭
   - GitHub 계정으로 로그인 (권장) 또는 이메일 회원가입

2-2. 새 프로젝트 만들기
   - "New Project" 클릭
   - 다음 정보 입력:
     * Organization: 기본 조직 사용 (또는 새로 생성)
     * Name: weather-app (또는 원하는 이름)
     * Database Password: 강력한 비밀번호 생성 (기억해두세요!)
     * Region: Northeast Asia (Seoul) 또는 가장 가까운 지역
     * Pricing Plan: Free 선택
   - "Create new project" 클릭
   - 프로젝트 생성 완료 대기 (1-2분)

2-3. API 키 확인 (중요!)
   - 프로젝트 대시보드에서
   - 좌측 메뉴 "Settings" → "API" 클릭
   - 다음 정보를 메모장에 복사해두기:
     * Project URL: https://xxxxx.supabase.co
     * service_role key: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... (매우 긴 문자열)
   - ⚠️ service_role key는 비공개이므로 안전하게 보관하세요!

2-4. 데이터베이스 스키마 생성 (중요!)
   - 좌측 메뉴 "SQL Editor" 클릭
   - "New query" 버튼 클릭
   - 프로젝트 폴더에서 "supabase/schema.sql" 파일 열기
   - 파일 전체 내용 복사 (Ctrl+A, Ctrl+C)
   - SQL Editor에 붙여넣기 (Ctrl+V)
   - 우측 하단 "Run" 버튼 클릭 (또는 Ctrl+Enter)
   - "Success. No rows returned" 메시지 확인
   - ⚠️ 이 단계를 건너뛰면 즐겨찾기 기능이 작동하지 않습니다!

2-5. 이메일 확인 비활성화 (중요!)
   - 좌측 메뉴 "Authentication" → "Settings" 클릭
   - "Enable email confirmations" 체크박스 해제
   - "Save" 클릭
   - ⚠️ 이 설정을 하지 않으면 회원가입 후 이메일 확인이 필요합니다!

완료 체크: □ Supabase 계정 생성됨
           □ 프로젝트 생성됨
           □ Project URL 복사함
           □ service_role key 복사함
           □ 데이터베이스 스키마 생성함
           □ 이메일 확인 비활성화함

5단계: 코드 준비 및 GitHub에 업로드
==========================================

5-1. 프로젝트 폴더 확인
   - 현재 폴더: c:\projects\weather-app
   - 이 폴더에 모든 파일이 있는지 확인

5-2. GitHub Desktop으로 업로드 (권장)
   
   방법 A: GitHub Desktop 사용
   1. GitHub Desktop 실행
   2. "File" → "Add Local Repository" 클릭
   3. "Choose..." 버튼 클릭
   4. c:\projects\weather-app 폴더 선택
   5. "Add repository" 클릭
   6. "Publish repository" 클릭
   7. 저장소 이름 확인: weather-app
   8. "Publish repository" 클릭

   방법 B: GitHub 웹사이트에서 직접 업로드
   1. GitHub 저장소 페이지 접속
   2. "uploading an existing file" 클릭
   3. weather-app 폴더의 모든 파일을 드래그 앤 드롭
   4. "Commit changes" 클릭

완료 체크: □ GitHub에 모든 파일 업로드됨
           □ api 폴더가 업로드되었는지 확인
           □ vercel.json 파일이 업로드되었는지 확인

6단계: Vercel에 배포
==========================================

6-1. 프로젝트 가져오기
   - Vercel 대시보드 접속
   - "Add New..." → "Project" 클릭
   - "Import Git Repository"에서
   - weather-app 저장소 선택
   - "Import" 클릭

6-2. 프로젝트 설정
   - Framework Preset: "Other" 선택
   - Root Directory: "./" (기본값)
   - Build Command: 비워두기
   - Output Directory: 비워두기

6-3. 환경 변수 설정 (중요!)
   - "Environment Variables" 섹션 클릭
   - 다음 3개 변수를 하나씩 추가:

   변수 1: SUPABASE_URL
     - Name: SUPABASE_URL
     - Value: (2단계에서 복사한 Project URL)
     - Environment: Production, Preview, Development 모두 체크
     - "Save" 클릭

   변수 2: SUPABASE_SERVICE_KEY
     - Name: SUPABASE_SERVICE_KEY
     - Value: (2단계에서 복사한 service_role key)
     - Environment: Production, Preview, Development 모두 체크
     - "Save" 클릭

   변수 3: OPENWEATHER_API_KEY
     - Name: OPENWEATHER_API_KEY
     - Value: (3단계에서 복사한 API 키)
     - Environment: Production, Preview, Development 모두 체크
     - "Save" 클릭

6-4. 배포 시작
   - "Deploy" 버튼 클릭
   - 배포 완료까지 2-3분 대기
   - "Congratulations!" 메시지가 나타나면 성공!

6-5. 배포 URL 확인
   - 배포 완료 후
   - "Visit" 버튼을 클릭하면 사이트가 열립니다
   - URL 예시: https://weather-app-xxxxx.vercel.app
   - 이 URL을 메모해두세요!

완료 체크: □ Vercel에 프로젝트 가져옴
           □ 환경 변수 3개 모두 설정함
           □ 배포 완료됨
           □ 배포 URL 확인함

==========================================
문제 해결
==========================================

Q: 배포가 실패합니다
A:
   1. Vercel → "Deployments" → 최신 배포 클릭
   2. "Build Logs" 확인
   3. 에러 메시지 확인
   4. 일반적인 에러:
      - "Module not found" → package.json 확인
      - "Environment variable" → 환경 변수 확인

Q: 회원가입/로그인이 작동하지 않습니다
A:
   1. Vercel 환경 변수 확인:
      - SUPABASE_URL이 올바른지 확인
      - SUPABASE_SERVICE_KEY가 service_role key인지 확인
   2. Supabase 설정 확인:
      - "Enable email confirmations" 해제 확인
   3. Vercel에서 "Redeploy" 실행

Q: 날씨 검색이 작동하지 않습니다
A:
   1. Vercel 환경 변수 확인:
      - OPENWEATHER_API_KEY가 올바른지 확인
   2. OpenWeatherMap에서 API 키 활성화 확인
   3. Vercel에서 "Redeploy" 실행

Q: 404 오류가 나타납니다
A:
   1. GitHub에 api 폴더가 있는지 확인
   2. GitHub에 vercel.json 파일이 있는지 확인
   3. Vercel에서 "Redeploy" 실행
