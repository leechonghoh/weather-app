==========================================
로컬 개발 서버 실행 방법
==========================================

현재 "서버 연결 오류"가 발생하는 이유:
- API 엔드포인트(/api/*)를 처리할 수 있는 서버가 실행되지 않았기 때문입니다.
- Python이나 Node.js의 간단한 HTTP 서버는 정적 파일만 제공하고 API를 처리하지 않습니다.

==========================================
해결 방법 1: Vercel CLI 사용 (권장)
==========================================

1단계: Node.js 설치 확인
   - PowerShell에서 다음 명령어 실행:
     node --version
   - 버전이 나오면 설치됨, 안 나오면 설치 필요
   - 설치: https://nodejs.org/

2단계: Vercel CLI 설치
   - PowerShell에서 다음 명령어 실행:
     npm install -g vercel

3단계: 프로젝트 폴더로 이동
   - PowerShell에서:
     cd c:\projects\weather-app

4단계: 의존성 설치
   - PowerShell에서:
     npm install

5단계: Vercel 개발 서버 실행
   - PowerShell에서:
     vercel dev
   - 또는:
     npm run dev:vercel

6단계: 브라우저에서 접속
   - Vercel이 제공하는 주소로 접속 (보통 http://localhost:3000)
   - 또는 터미널에 표시된 주소 사용

==========================================
해결 방법 2: Express 개발 서버 사용
==========================================

1단계: Node.js 설치 확인 (위와 동일)

2단계: 프로젝트 폴더로 이동
   - PowerShell에서:
     cd c:\projects\weather-app

3단계: 의존성 설치
   - PowerShell에서:
     npm install

4단계: 개발 서버 실행
   - PowerShell에서:
     npm run dev
   - 또는:
     node dev-server.js

5단계: 브라우저에서 접속
   - http://localhost:3000

==========================================
환경 변수 설정 (.env 파일)
==========================================

프로젝트 루트에 .env 파일을 생성하고 다음 내용을 추가하세요:

SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_KEY=your_supabase_service_key
OPENWEATHER_API_KEY=your_openweather_api_key

==========================================
문제 해결
==========================================

Q: "npm을(를) 인식할 수 없습니다" 오류
A: Node.js가 설치되지 않았습니다. https://nodejs.org/ 에서 설치하세요.

Q: "포트가 이미 사용 중입니다" 오류
A: 다른 포트를 사용하거나, 해당 포트를 사용하는 프로그램을 종료하세요.

Q: 여전히 "서버 연결 오류"가 발생합니다
A: 
   1. 서버가 실행 중인지 확인하세요
   2. 브라우저 개발자 도구(F12) > Console 탭에서 에러 확인
   3. 서버 콘솔에서 에러 메시지 확인
   4. .env 파일이 올바르게 설정되었는지 확인
